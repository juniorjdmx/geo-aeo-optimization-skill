# Contributing

Thanks for taking the time to contribute.

## Quick start

1. Fork the repo and create a feature branch.
2. Make your changes with clear, focused commits.
3. Ensure docs and templates remain in English and links work.
4. Open a Pull Request describing what you changed and why.

## Style & scope

- Keep changes minimal and relevant.
- Prefer simple, readable Markdown.
- Do not add private data or secrets.

## Reporting issues

If you find a bug or want an improvement, please open an issue with:
- what you expected vs what happened
- steps to reproduce
- screenshots/logs if relevant
